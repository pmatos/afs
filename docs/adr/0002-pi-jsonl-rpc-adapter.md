# Speak Pi's JSONL JSON-RPC and Pin Structured Output Through a Vendored Extension

AFS speaks Pi's documented JSONL JSON-RPC over child stdio (per `@mariozechner/pi-coding-agent/docs/rpc.md`) and pins structured output through a vendored `afs_reply` Pi extension whose TypeBox `parameters` declare the schema every directory-agent turn must satisfy. We chose this after the v1 tab-delimited protocol turned out to be defined by AFS's in-tree shell-script fake and was silently rejected by real `pi --mode rpc` (issue #40); the supervisor seam (subprocess RPC) is preserved, but the wire format is now what Pi actually implements and the structured-output contract is enforced by Pi's own tool-call validation rather than by free-form text parsing.
